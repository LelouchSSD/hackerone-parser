# HackerOne Researcher Scraper (Community Research Tool)

## Описание проекта
Инструмент для автоматизированного сбора, структурирования и анализа данных о багхантерах с платформы HackerOne. Проект разработан в рамках тестового задания на позицию **Community Research Engineer** в Positive Technologies.

Основная цель - выявление наиболее активных и ценных исследователей для привлечения на платформу Standoff Bug Bounty.

## Ключевые особенности
* **GraphQL Interception:** Скрипт не просто парсит HTML, а перехватывает прямые ответы GraphQL API, что обеспечивает высокую точность и полноту данных.
* **Обход ограничений:** Использование Playwright для эмуляции поведения пользователя и обхода защитных механизмов (Cloudflare).
* **Умное слияние данных:** Реализована логика рекурсивного слияния (`deep_merge`) для обновления профилей без потери уникальных данных (бейджы, специализации).
* **Аналитика экспертизы:** Извлечение метрик качества (Signal, Impact) и векторов специализации (Weakness Stats).

## Стек технологий
* **Language:** Python 3.13
* **Automation:** Playwright (Chromium)
* **Data Format:** JSON (структурированное хранение профилей)
* **Version Control:** Git

## Установка и запуск

### Клонирование репозитория
```bash
git clone [https://github.com/LelouchSSD/hackerone-parser.git](https://github.com/LelouchSSD/hackerone-parser.git)
cd hackerone-parser
```
### **Настройка виртуального окружения**
```bash
python -m venv venv
source venv/bin/activate  # Для Linux/macOS
venv\Scripts\activate     # Для Windows
```
### Установка зависимостей
```bash
pip install -r requirements.txt
playwright install
```
### Запуск
```
python main.py
```
## Структура данных

Результаты сбора сохраняются в директорию `researchers_info/`:
- `user/` - папка конкретного пользователя.
	- `highlights.json` - агрегированная и очищенная информация (уровень, баунти, бейджи, специализация).
    - `response_X.json` - сырые GraphQL-ответы для возможности последующего переанализа.
## Направления развития (Roadmap)

1. **Масштабируемость:** Переход от хранения в JSON к базе данных для сложной аналитики.
2. **Анонимность:** Интеграция пула прокси-серверов и систем ротации.
3. **Предиктивная аналитика:** Разработка модели оценки вероятности перехода хакера на платформу Standoff на основе его текущей активности.
